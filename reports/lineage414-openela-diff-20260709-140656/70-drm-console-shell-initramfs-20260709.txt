# Downstream DRM console and visible command shell

Date: 2026-07-09 21:57 CEST

## Result

The downstream Lineage/OpenELA 4.14.357 pmOS boot now has a reusable visible
text path on the OnePlus 7T Pro panel.

The tested boot image injects a small DRM/KMS console helper into the initramfs,
starts it when `/dev/dri/card0` appears, stops that initramfs instance before
`switch_root`, and lets pmOS userspace start the persistent visible command
shell.

```text

sha256: 1075757fe6c7a582b94c4a9f837cd71b830d36da8e29c60acba85c49e6c57019

boot_id: 7854ea12-7415-41bc-8f2e-59d8865fd041
```

## Boot evidence

The device returned to pmOS SSH after the test reboot. The kernel log contains:

```text

[    2.568984] [hotdog-watchdog] marker: usb-iface
[    5.570778] [hotdog-watchdog] marker: root-mounted
[    5.651583] [hotdog-watchdog] marker: switch-root
```

The visible userspace command shell is controlled by:

```text

transcript: /tmp/hotdog-drm-console.transcript
```

Command-channel evidence:


## Implementation

Tracked host-side pieces:

```text
helpers/hotdog-drm-console.c
scripts/install-hotdog-drm-console.sh
scripts/build-watchdog-bootimg.sh --drm-console
```

Generated local helper binary:

```text
/home/srobin/dev/hotdog/build/hotdog-drm-console-aarch64
sha256: 4410e744b5b73b727fd039a507ba1ebc728b17ac75198d8be93a90b1d50ade09
```

The binary above is a generated local artifact, not a source substitute. A fresh
host should rebuild it from `helpers/hotdog-drm-console.c` or pull a new copy
from a booted pmOS system after compiling it there.

## Interpretation

This is the first validated boot cycle that gives both:

1. visible text on the phone panel before pmOS userspace is fully up
2. a post-boot visible root shell controlled from the host

That changes the next mainline strategy. USB gadget reachability is no longer
the only early milestone; a useful mainline candidate should try to bring up
enough DRM/KMS, framebuffer, or panic text to make the failure visible even if
USB never returns.

## Follow-up

- Keep the older downstream `195300` image as the rescue restore target.
- Keep this `215005` DRM-console image as the current visible diagnostic
  baseline.
- Make the helper build reproducible on a fresh host so `--drm-console` does
  not depend on a local generated binary.
- When retesting mainline, prefer watchdog/pstore plus visible-console
  instrumentation over the old `/dev/fb0` paint test.
