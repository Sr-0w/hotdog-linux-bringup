Purpose:

  044715 supersedes 045000 as the automatic next hardware test. It keeps:

    - pmaports lineage414 kernel r2
    - fixed entry12 /chosen ranges; DTB pack
    - initramfs tty-kmsg/fbprep follower
    - DRM console userspace hook
    - rootfs visible tty auto-cycle pages every 12s
    - sysrq-first watchdog timeout reboot

  The new behavior is stricter USB-success detection. The watchdog no longer
  treats a configured configfs gadget/UDC as success. It now requires one of the
  USB network interfaces to show carrier=1, or operstate=up when carrier is not
  available.

Why:

  The observed failure has screen text but no ADB, fastboot, SSH, or host USB
  device. Peirce's read-only audit confirmed that the previous USB mode could
  still succeed on a non-empty /sys/kernel/config/usb_gadget/g1/UDC. That can
  leave the phone in a rootfs state that is useful visually but not commandable
  from the host. This candidate preserves the visual diagnostics but should
  still reboot if USB carrier never appears.
