Display
-------

Lineage DTBO contains the real downstream panel/display data, including many
`qcom,dsi-display@*` entries and active OnePlus/Samsung panel selections. The
useful known identifiers include:

```text
dsi_samsung_oneplus_dsc_cmd_display
dsi_samsung_s6e3fc2x01_cmd_display
dsi_samsung_sofef03f_m_fhd_dsc_cmd_display
```

Those nodes require the downstream SDE/DSI/panel stack. They are not a minimal
screen-debug path for mainline.

For early visual diagnostics, keep using bootloader-preserved simplefb first:

```text
framebuffer@9c000000
compatible = "simple-framebuffer"
width = 1440
height = 3120
stride = 5760
format = "x8r8g8b8"
stdout-path = "display0"
```

This is testable in the DTB pack only, provided the kernel has `CONFIG_VT`,
`CONFIG_FB_SIMPLE`, and `CONFIG_FRAMEBUFFER_CONSOLE`.

4. Treat Lineage DTBO display nodes as later real-display bring-up material, not
