Passive host USB/dmesg watcher hardening
========================================

Date: 2026-07-10 05:09 CEST

Context:

  The current phone boot shows visible text on the panel, but the host still has
  no ADB, fastboot, postmarketOS SSH, OnePlus USB, Qualcomm EDL, or Android USB
  device. In that state, `lsusb` alone cannot distinguish "nothing happened"
  from repeated host-side USB reset/enumeration failures.

Changes:

  - watch-phone-state.sh now stores a filtered host kernel USB log as
    dmesg-usb-interesting.txt beside each snapshot.
  - latest-summary.txt includes a `dmesg_usb=` block when matching lines exist.
  - snapshot_key now includes those filtered dmesg lines, so a new snapshot is
    kept when the kernel logs USB resets/disconnects/descriptors even if adb,
    fastboot, and lsusb stay unchanged.
  - current-autopilot-status.sh now matches the detached passive watcher by the
    generic `watch-phone-state.sh --timeout 21600 --poll 5` command suffix, so
    the process list no longer reports a false `passive-phone-state missing`
    when the watcher was launched from a relative path.

Why:

  This keeps the watcher passive and non-invasive, but makes it useful during
  black-screen or visible-text boots where the phone never reaches a host-visible
  USB function. It gives the next person resuming the project one place to check
  for host kernel evidence before deciding whether to ask for physical
  interaction.
