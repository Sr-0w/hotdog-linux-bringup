drm-console userspace candidate - 2026-07-10
================================================

Goal
----

Make the validated downstream 4.14 path more reproducible as a screen-side
debug baseline. The previous `215005` image proved that the injected initramfs
DRM console can show text on the OnePlus 7T Pro panel, and the live rootfs had
a userspace DRM command shell available through `/tmp/hotdog-drm-console.in`.
This follow-up removes the dependency on that userspace hook having already
been installed by hand.

Builder change
--------------

`scripts/build-watchdog-bootimg.sh` now supports:

```text
--drm-console-userspace
```

The option implies `--drm-console`. During rootfs postmount, before
`switch_root`, it copies the injected `hotdog-drm-console` binary and PSF font
into `/sysroot`, writes `/sysroot/etc/local.d/hotdog-drm-console.start`, and
ensures OpenRC `local` is present in the default runlevel when the rootfs has
the expected layout.

Why this is not kernel-early text yet
-------------------------------------

The downstream package and DTB pack have the right pieces on paper:
`CONFIG_FB_SIMPLE=y`, `CONFIG_FRAMEBUFFER_CONSOLE=y`, `console=tty0`, and a
patched DTB-pack entry with `simple-framebuffer` geometry plus stdout-path.
The runtime evidence still points to a failed simplefb handoff:

```text
/dev/fb0 is absent
simple-framebuffer chosen:framebuffer@9c000000: No memory resource
```

So the reliable current screen path is DRM/KMS after `/dev/dri/card0` appears.
Getting printk on the panel before initramfs probably requires fixing the
simplefb memory resource / fbcon attach path, or rebuilding/patching the
downstream kernel. More cmdline-only variants are unlikely to be enough.

Rescue watcher robustness
-------------------------

`scripts/rescue-boot-b-when-visible.sh` now supports file-only logging through:

```text
HOTDOG_RESCUE_LOG_TEE=0
```

The companion launchers set that environment for detached rescue watchers:

```text
scripts/test-boot-b-image.sh
scripts/start-stable-rescue-watcher.sh
```

Current limitation
------------------

The `005100` image has not been flashed yet. The phone is currently absent from
fastboot, ADB, and `lsusb` after the mainline 6.17 stock-DTB-pack timeout. Wait
for fastboot/recovery ADB or a manual reset before testing this candidate.

Superseding note
----------------

After this image was prepared, offline DTB inspection found a likely reason for
the simplefb/fbcon failure: the later multi-DTB entry12 simplefb node was
missing `ranges;` under `/chosen`. The next priority test is therefore the
newer `010900` image from report `75`, which includes this userspace console
work plus a fixed entry12 DTB pack and `--fb-test`.
