drm command shell candidate - 2026-07-10
========================================

Reason
------

The display path now produces visible text, but the previous initramfs DRM
console command used a foreground `while :; dmesg; sleep 2; done` loop. That is
useful as a text-output proof, but it keeps `/bin/sh` busy and makes the FIFO
input path look like "not really a shell".

Code changes
------------

`scripts/build-watchdog-bootimg.sh` now starts the initramfs DRM console with a
one-shot boot snapshot:

```text
--- hotdog initramfs DRM command shell ---
stage: ...
cmdline: ...
--- initial dmesg snapshot ---
...
--- ready: commands are read from /tmp/hotdog-drm-console.in ---
initramfs#
```

The userspace DRM console path is unchanged. It still uses
`/tmp/hotdog-drm-console.in` and `/tmp/hotdog-drm-console.transcript`.

`scripts/build-hotdog-drm-console-helper.sh` now rebuilds the DRM helper from
`helpers/hotdog-drm-console.c` through the pmbootstrap aarch64 buildroot.
`scripts/build-watchdog-bootimg.sh` calls this automatically when `--drm-console`
is enabled and the default helper binary is missing:
