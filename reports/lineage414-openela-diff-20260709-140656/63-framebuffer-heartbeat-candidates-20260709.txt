2026-07-09 framebuffer heartbeat candidates
===========================================

Why this exists
---------------

The mainline 6.17 full-DTB-pack test no longer returned immediately to
fastboot, but it also produced no USB, SSH, telnet, adb, or visible console.
That leaves two competing explanations:

- the kernel reaches initramfs but console binding/fbcon does not show text;
- the kernel hangs earlier than initramfs/userspace.

The next diagnostic should separate those cases on the downstream 4.14 kernel
that is already known to boot pmOS SSH. A raw /dev/fb0 paint test is more direct
than relying on fbcon text. If simplefb is alive in initramfs, it should paint
red, green, blue, then white even if the Linux console never binds correctly.

Builder change
--------------

Script:

  /home/srobin/dev/hotdog/scripts/build-watchdog-bootimg.sh

New option:

  --fb-test

Effect:

- injects /hotdog_fb_test.sh into the initramfs;
- starts it before dynamic partition/root discovery in init_2nd;
- waits up to 45 seconds for /dev/fb0 or /dev/graphics/fb0;
- computes framebuffer bytes from /sys/class/graphics/fb0 when available;
- falls back to 1440x3120x32bpp;
- paints red, green, blue, then white;
- logs as [hotdog-fb-test] to kmsg and stderr.

Validated with:

  busybox ash -n hotdog_fb_test.sh
  busybox ash -n hotdog_rescue_watchdog.sh
  busybox ash -n init
  busybox ash -n init_2nd.sh

Interpretation matrix
---------------------

- Screen paints colors, then SSH returns:
  simplefb works and root boot path survived; future mainline debugging can use
  the same early framebuffer technique.

- Screen paints colors, no SSH:
  kernel reached initramfs and simplefb works; root discovery, USB gadget, or
  switch-root needs investigation.

- No colors, but SSH returns:
  downstream still boots, but simplefb/fb0 binding is not alive; inspect
  /proc/fb, /sys/class/graphics, dmesg simplefb/fbcon, and pstore.

- No colors, no USB, and no fastboot return:
  either the kernel hangs before initramfs or the bootloader framebuffer is not
  exposed as Linux fb0 even with the DTB instrumentation.

- Immediate fastboot return:
  bootloader rejected the image or slot state changed; restore boot_b from the
  stable image and inspect fastboot vars.
