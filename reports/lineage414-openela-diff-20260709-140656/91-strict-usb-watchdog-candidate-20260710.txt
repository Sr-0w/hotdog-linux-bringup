Purpose:

Promote the `034500` screen/button diagnostics to the automatic next hardware
test, but avoid the root-success watchdog weakness. The previous `034500`
candidate intentionally used `--watchdog-success root`, which is useful for
long observation when rootfs boots, but it can leave the phone in a visible
rootfs state without host USB/SSH. The `042500` candidate keeps the same kernel,
DTB pack, tty-kmsg/fbprep hooks, DRM helper, and rootfs button tty shell, while
requiring a USB gadget/network success marker to stop the watchdog.

Builder hardening:

```text
--watchdog-success usb now means USB-only:
  - /tmp/hotdog_rescue_watchdog.ok still overrides intentionally
  - usb/rndis/enx/eth network interface succeeds
  - non-empty /sys/kernel/config/usb_gadget/g1/UDC succeeds
  - bare /sys/class/udc/* controller presence no longer succeeds

watchdog timeout action:
  - sync
  - sysrq reboot via /proc/sysrq-trigger b
  - fallback reboot -f
  - fallback sysrq panic
```
